1. Наследование -  это механизм, позволяющий классу наследовать свойства и методы другого класса. Класс, который наследует свойства, называется подклассом, а класс, от которого наследуются свойства, называется суперклассом.

В Java наследование реализуется с помощью ключевого слова "extends" в коде. При этом подкласс получает доступ ко всем открытым свойствам и методам суперкласса, а также может переопределить их или добавить новые.

2.Преимущеста:
Класс может наследовать свойства и методы сразу от нескольких суперклассов, из за чего можно избежать дублирования кода.
Множественное наследование позволяет создавать более сложные иерархии классов, что упрощает проектирование и разработку программ.

Недостатки:
Если два суперкласса имеют методы с одинаковыми именами, то может возникнуть неоднозначность, когда подкласс вызывает этот метод.
Множественное наследование может привести к усложнению структуры классов и ухудшению читаемости кода.

3. ООП - это способ написания кода, где мы работаем с объектами. Объект - это что-то, у чего есть данные и действия, которые можно с ними совершать. Программы строятся вокруг этих объектов.

Полиморфизм - это возможность использовать один и тот же код для разных типов данных или объектов.

4. В Java полиморфизм с использованием методов реализуется через переопределение методов. Если подкласс переопределяет метод суперкласса, объект этого подкласса может использоваться вместо объекта суперкласса, и вызов метода будет направлен к версии метода в подклассе.

5. Статический полиморфизм - это как использование одного имени для разных задач в одном месте программы.
Динамический полиморфизм - это, когда объекты разных типов ведут себя по-разному при выполнении одной и той же операции
